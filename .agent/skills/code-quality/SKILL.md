# 💎 GitData Code Quality

Standardized rules for maintaining a premium and readable codebase.

## 📏 General Rules
1. **No Comments**: Code must be self-explanatory. Use descriptive names instead of comments.
2. **Remove Dead Code**: Never leave commented-out code blocks in components.
3. **Self-Documenting Props**: Use TypeScript interfaces to make component usage clear.
4. **Consistency**: Use the same patterns for state management (Pinia) and data fetching across the app.
