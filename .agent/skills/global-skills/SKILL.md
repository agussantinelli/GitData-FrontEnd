# 📚 GitData FrontEnd Skills Catalog

This skill serves as a central index for all architectural and development guidelines established in the GitData FrontEnd project. Each skill defines rules and best practices that **MUST** be respected by the agent and developers.

## ⚡ Core Skills (`GitData-FrontEnd`)

- **[vue3-primevue](file:///c:/Users/Agus/Documents/GitData/GitData-FrontEnd/.agent/skills/vue3-primevue/SKILL.md)**: Mandatory rules for Vue 3 (Composition API), PrimeVue 4, and Neon Styling.
- **[testing-strategy-fe](file:///c:/Users/Agus/Documents/GitData/GitData-FrontEnd/.agent/skills/testing-strategy-fe/SKILL.md)**: Guidelines for frontend testing using Vitest and Vue Test Utils.
- **[code-quality](file:///c:/Users/Agus/Documents/GitData/GitData-FrontEnd/.agent/skills/code-quality/SKILL.md)**: Guidelines for clean code, readability, and the "No Comments" policy.
- **[responsive-design](file:///c:/Users/Agus/Documents/GitData/GitData-FrontEnd/.agent/skills/responsive-design/SKILL.md)**: Mobile-first and fluid layout best practices for the GitData Dashboard.
- **[readme-auto-sync](file:///c:/Users/Agus/Documents/GitData/GitData-FrontEnd/.agent/skills/readme-auto-sync/SKILL.md)**: Mandatory automated README syncing for project structure and tests.
- **[test-enforcement](file:///c:/Users/Agus/Documents/GitData/GitData-FrontEnd/.agent/skills/test-enforcement/SKILL.md)**: Mandatory rule requiring corresponding tests for every business logic file and UI component.
- **[skill-generator](file:///c:/Users/Agus/Documents/GitData/GitData-FrontEnd/.agent/skills/skill-generator/SKILL.md)**: A tool for creating new skills following the GitData standard.

## 🛠️ Usage guidelines

1. **Check before coding**: Always consult the relevant skill(s) before implementing a new feature.
2. **Consistency over speed**: If a skill defines a pattern, follow it strictly.
3. **Skill updates**: When a new architectural pattern is established, update the relevant skill or create a new one using the `skill-generator`.
